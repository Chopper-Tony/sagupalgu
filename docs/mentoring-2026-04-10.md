# 사구팔구 프로젝트 — 멘토링 발표 자료 (2026-04-10)

## 프로젝트 개요

중고거래(번개장터/중고나라) 자동 게시 플랫폼. 이미지 업로드만 하면 AI가 상품 분석 → 시세 산정 → 판매글 생성 → 자동 게시까지 수행.

- **기술 스택**: FastAPI + LangGraph + React + Chrome Extension + Playwright
- **아키텍처**: 7 에이전트 / 10 툴 / 3 Agentic Loop
- **테스트**: 723개 통과

---

## 이번 주 (4/8~4/9) 주요 작업

### 1. 게시 아키텍처 대전환 — 서버 Playwright → 크롬 익스텐션 Content Script

**배경**:
- **번개장터 계정 영구정지** — 미국 EC2 IP에서 게시 → 한국 로그인과 지역 불일치로 계정 탈취 패턴으로 탐지
- **중고나라 CloudFront 403** — AWS IP 대역 자체를 봇 방지로 차단

**해결**: 서버에서 게시하지 않고, 크롬 익스텐션의 Content Script가 사용자 브라우저에서 직접 폼을 채우는 방식으로 전환. 사용자 IP/쿠키/핑거프린트 그대로 사용해서 플랫폼이 자동화를 탐지할 수 없음.

**핵심 난관 — 이미지 업로드**:
- 브라우저 보안 정책상 JavaScript로 `input[type=file]`에 프로그래밍 파일 주입 불가
- 5번 시도 끝에 **CDP(Chrome DevTools Protocol) + Runtime.evaluate + React fiber onChange 직접 호출** 방식으로 해결
- Playwright가 내부적으로 쓰는 방식과 동일한 원리

### 2. 게시 UX 통합
- 초기: 세션 ID 복사 → 익스텐션 팝업 붙여넣기 → 게시 버튼 (3단계)
- 개선: 웹앱에서 "자동 게시" 버튼 클릭 → `window.postMessage` → `publish_bridge.js` → 익스텐션 자동 실행 (1단계)

### 3. 자체 마켓 페이지 구현
- 플랫폼 종속성에서 벗어나기 위한 **자체 중고거래 마켓** 프로토타입
- `/market` 페이지: 등록된 상품 카드 그리드
- **상품 상세 페이지**: 이미지 갤러리 + 플랫폼 게시글 바로가기
- **검색 + 필터**: 키워드 검색 (디바운스 300ms) + 가격 범위
- **구매 문의**: Discord 웹훅으로 판매자에게 알림

### 4. 인프라/보안
- **Supabase RLS 활성화** (보안 경고 대응)
- **EBS 19GB 증설** (Docker 빌드 정상 복구)
- **프론트 UX 개선 7건** — 에러 메시지, 필드 검증, 로딩 상태, textarea 자동 확장 등
