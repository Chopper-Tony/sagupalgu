문제 정의

기존 중고거래 판매 과정

  • 상품 사진 촬영
  • 시세 직접 조사 (여러 플랫폼 검색)

게시 아키텍처 대전환

Before: 서버 Playwright

  • EC2(미국)에서 headless Chrome 실행
  • 서버 IP → 번개장터 계정 영구정지
  • AWS IP 대역 → 중고나라 CloudFront 403
  • Chromium 메모리 1.5GB 점유

After: Chrome Extension Content Script

  • 사용자 브라우저에서 직접 게시
  • 사용자 IP/쿠키/핑거프린트 그대로
  • 자동화 탐지 거의 0
  • 서버 메모리 부담 제거
핵심 난제 해결: 이미지 업로드 — 브라우저 보안상 input[type=file] 프로그래밍 주입 불가
→ CDP Runtime.evaluate + base64→File + React fiber onChange 직접 호출
